## 一、项目概述

基于大语言模型与检索增强生成技术，打造 AI 驱动的交互式名著导读平台。首发《西游记》，后续扩展至三国演义、红楼梦、水浒传。

用户通过自然语言提问，系统从知识库检索原文及解析资料，结合 LLM 生成准确、有据可依的回答。

## 二、功能需求

### 2.1 已交付

| 功能 | 说明 |
|------|------|
| 知识库检索问答 | 四部名著全文 + 人物 + 情节 + 评论，BCE Embedding + Reranker，Top 4 召回 |
| 条件分支路由 | 检索命中→严谨问答；未命中→学者风格半文半白 |
| 自定义前端 | React 双栏布局（左栏原著阅读 + 右栏 AI 对话） |
| 章节阅读 | GET /api/chapters，支持四部名著 100+ 回原文检索（DB 查询） |
| 全文搜索 | MySQL FULLTEXT 索引，支持跨书关键词搜索章节 |
| 推荐问题引导 | 5 条推荐问题一键提问 |
| 用户反馈机制 | 赞/踩按钮 |
| 多作品切换 | 西游记/三国演义/红楼梦/水浒传四部全部开通 |
| 对话持久化 | localStorage 按作品分组存储，刷新不丢失 |
| 多轮对话上下文 | conversation_id 传递，连续提问理解前文 |
| 阅读进度记忆 | 服务端持久化（reading_history 表），支持继续阅读 |
| API Key 安全 | 前端不再硬编码，后端环境变量注入 |
| TTS 原文朗读 | Web Speech API、播放/暂停/停止、段落高亮、进度条拖动、文本跟随滚动 |
| 深色模式 | 跟随系统 / 手动切换，CSS 变量主题 |
| 移动端适配 | 汉堡菜单 + overlay 侧栏，触屏响应式布局 |
| 管理后台 | 用户列表（分页）、系统统计、健康监控 |
| FastAPI 后端 | http.server → FastAPI + Uvicorn + Pydantic 校验 |
| API 限流 + 安全 | slowapi 6 端点分级限流、安全响应头、登录失败锁定 |
| Docker 部署 | Docker + docker-compose + Caddy 反向代理 |
| CI/CD | GitHub Actions（lint + test + build + docker） |
| 自动化备份 | 数据库 + data 目录备份 + Windows 计划任务 |
| 外网访问 | cpolar 隧道 |
| Android App | WebView 封装 APK |

## 三、技术栈

| 类别 | 选型 |
|------|------|
| AI 平台 | Dify 0.6.0 |
| 大语言模型 | DeepSeek V4 Flash |
| Embedding | BCE Embedding Base (netease-youdao) |
| Reranker | BCE Reranker Base (netease-youdao) |
| 模型服务 | SiliconFlow |
| 后端 | FastAPI + Uvicorn（V3.0.0） |
| 前端 | React 19 + Vite |
| 数据库 | MySQL 8.0（用户 + 章节全文索引） |
| Android | WebView 封装 |
| 反向代理 | Caddy（安全头 + 内网路由） |
| 公网隧道 | cpolar |
| 容器化 | Docker + docker-compose |
| CI | GitHub Actions |
